WUI: A Complete Control Web CMS

Introduction

The idea behind WUI is simple:

  • You have a html file and you want to publish it to the internet.
  • You want to group your html files by menu-driven interface.
  • Let the user (browser) do the process, server only provided the data and the script.
  • Database-less, running without need of database service.
  • Complete control of your site.

WUI is a hack to HTML meta tag, HTML DOM, and JavaScript to create a semi-dynamic website.

Design

WUI front page is divided by five groups: header, menu, info, content, and footer. Except for menu and info groups, each group has its own "html" source.

+-------------------------------------------------------------------------+
| HEADER                                                                  |
| (static HTML file)                                                      |
+-------------------------------------------------------------------------+
| MENU                                                                    |
| (created by JS using data from wui-menu.js)                             |
+-------------------------------------------------------------------------+
| INFO                                                                    |
| (created by JS using meta tag from content HTML file)                   |
+-------------------------------------------------------------------------+
| CONTENT                                                                 |
| (static HTML file, displayed using div element)                         |
+-------------------------------------------------------------------------+
| FOOTER                                                                  |
| (static HTML file)                                                      |
+-------------------------------------------------------------------------+

Experienced web developer can extend this group into more section or modified it based on their need.

All of group source or html file in WUI is using the same style: wui.css.

All of interface and control is handled using one JavaScript: wui.js

Header Group

Header group is provided by header.html file, located in the root directory.

Menu Group

All WUI menu is auto-generated by script: scripts/create_index.rb

When the script is running, it will find a directory recursively that have index.html file. If content of index.html file does not have a body tag then the directory itself will become a menu.

The script will generate one output file: wui-menu.js.

Info Group

Info group is a section in front page that will displayed all meta tag in html file, as additional information what the content is about.

Content Group

Content group is a section in front page that will displayed the content of the site (HTML file).

Footer Group

Footer group is provided by footer.html file, located in the root directory.

Deployment Tutorial

  • Create a local document root for your site.
    In example: www
  • Copy WUI source code to the document root directory.
  • Modify header.html and/or footer.html based on your requirements.
  • Run a HTTP server using www as document root

Creating a page and menu

Inside document root www create another directory, and inside those directory create a file named index.html. Each of index.html must have meta tag that contain at least title. In example,

<html>
<head>
<meta name='title' content='a name for menu or title of content'/>
</head>

In this example we will create one page and one page inside of sub-menu.

To create a page with name is "About" with content,

  • Create a directory named "about".

    The name of directory does not matter, it can be anything. The one that affect the name of menu in your site is the content attributes in your meta tag in index.html file.

  • Inside of that directory create file index.html, which content is 
    <html>
<head>
<meta name='title' content='About'/>
<meta name='publish-date' content='2010.10.04 14:48:25 +0700'/>
<meta name='author' content='me@site.org'/>
<link rel='stylesheet' href='/wui.css' type='text/css'/>
</head>

<body>
<h1><a href='./index.html'> About </a></h1>
This is a content of About page.
</body>
</html>

To create a sub-menu "Profile" inside of menu "Me":

  • Create a directory "me" in top of document root.

  • Inside of that directory create file index.html, which content is

    <html>
<head>
<meta name='title' content='Me'/>
<meta name='publish-date' content='2010.10.04 14:50:00 +0700'/>
<meta name='author' content='me@site.org'/>
<link rel='stylesheet' href='/wui.css' type='text/css'/>
</head>
</html>

    If this menu does not display any content, you should not add body element to the HTML file.

  • Create a directory "profile" inside of "me" directory.

  • Inside of that directory create file index.html, which content is

    <html>
<head>
<meta name='title' content='Profile'/>
<meta name='publish-date' content='2010.10.04 14:50:00 +0700'/>
<meta name='author' content='me@site.org'/>
<link rel='stylesheet' href='/wui.css' type='text/css'/>
</head>

<body>
<h1><a href='./index.html'> Profile </a></h1>
This is a content of Profile page.
</body>
</html>

Testing and Preview

Now that you have already create some pages and/or probably with sub-menu, it's time to generate the index of pages and menus.

  • Go to the document root, www

  • Run the script from the www directory,

     $ ./scripts/create_index.rb

    This script will create file wui-menu.js in document root directory.

    If the script print an error, that mean is one of your HTML file is not well formatted. Fix your HTML file by looking at error line number. Example of error code:

    Missing end tag for 'p' (got "body")
Line: 266
Position: 6994

    NOTE: Remember that, every time you add or remove a new directory with index.html file you must run this script to generate a new wui-menu.js file.

  • Open and view your local site from web browser.

That's it. At this point you probably have an idea how the WUI work internally and you can extend it by your self.
